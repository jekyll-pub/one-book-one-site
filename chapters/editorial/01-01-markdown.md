# 1.1 Markdown 基础

Markdown 是一种简单易学的标记语言。主要分为两部分内容：字级标记和块级标记。两者的区别是字级标记文字可以放在一行以内，而块级标记文字是不可能放到一行之内的。

{: .note }
使用 Markdown 标记过程中，所有的标记符号都要使用英文半角状态下的字母符号。使用其他符号可能导致标记失效。

## 1.1.1 字级标记

### 1.1.1.1 突出文字显示

这种标记用来标记单个字词。例如想突出显示“重要”这两个字，可以这样标记：

这句话很*重要*！
{: .code-example}

实现代码：

~~~ markdown
这句话很*重要*！
~~~

{: .warning}
上面这种利用斜体突出显示文字的方法在中文中并不存在，只是西文的一种表达方式。其实，我们可以将其改为文字下面加点的方式显示。

另外，还可以通过加粗文字显示其重要性：

这句话很**重要**！！
{: .code-example}

实现代码：

~~~ markdown
这句话很**重要**！！
~~~

如果，想表示顶级重要，可以叠加使用：

这句话很***重要***！！！
{: .code-example}

实现代码：

~~~ markdown
这句话很***重要***！！！
~~~

### 1.1.1.2 链接和图像

#### 链接

链接通常由链接显示文字和链接地址两部分构成。标记方法是将链接文字放入方括号中，紧跟其后就在圆括号中放入链接地址，一个链接就标记完成。例如：

[中国国家图书馆](https://nlc.cn)拥有丰富的藏书。
{: .code-example}

实现方法：

~~~ markdown
[中国国家图书馆](https://nlc.cn)拥有丰富的藏书。
~~~

可以增加链接的提示信息：

[中国国家图书馆](https://nlc.cn "中国国家图书馆官方站点")拥有丰富的藏书。
{: .code-example}

实现方法：

~~~ markdown
[中国国家图书馆](https://nlc.cn "中国国家图书馆官方站点")拥有丰富的藏书。
~~~

当把鼠标指针悬停到链接上时就会显示提示信息。

还有一种尽量不打断文字流的链接创建方式——将链接的 URL 和 提示标题用方括号括起来的参考关键词的方式使用，而对参考关键词的定义只需要在文字最后定义即可：

[中国国家图书馆][国图]拥有丰富的藏书。
{: .code-example}

[国图]: https://nlc.cn "中国国家图书馆官方网站"

实现方式：

~~~markdown
[中国国家图书馆][国图]拥有丰富的藏书。

[国图]: https://nlc.cn "中国国家图书馆官方网站"
~~~

或者，干脆直接让链接文字和参考关键词相同：

[中国国家图书馆]拥有丰富的藏书。
{: .code-example}

[中国国家图书馆]: https://nlc.cn "中国国家图书馆官方网站"

实现方式：

~~~markdown
[中国国家图书馆]拥有丰富的藏书。

[中国国家图书馆]: https://nlc.cn "中国国家图书馆官方网站"
~~~

#### 图像

图像的使用方法跟链接很相似，只需要在链接前面加一个叹号即可：

这个图像上是![青草](../../../assets/images/grass.png)
{: .code-example}

实现方式：

~~~markdown
这个图像上是![青草](../../../assets/images/grass.png)
~~~

也可以加上提示标题：

这个图像上是![青草](../../../assets/images/grass.png "青青河边草")
{: .code-example}

实现方式：

~~~markdown
这个图像上是![青草](../../../assets/images/grass.png "青青河边草")
~~~

在图像上停留鼠标指针会出现提示。

### 1.1.1.3 行内代码

行内代码使用反引号`` ` ``包围来表示：

代码 `int a = 3` 表示 `a` 的值为 3
{: .code-example}

实现方式：

~~~markdown
代码 `int a = 3` 表示 `a` 的值为 3
~~~

如果代码中需要反引号怎么办呢？只需要至少两个反引号和空格包围就可以了：

代码标识方式是 `` `code` ``
{: .code-example}

实现方式：

~~~markdown
代码标识方式是 `` `code` ``
~~~

### 1.1.1.4 脚注

脚注就是通过每页地脚（页面最下端）的位置对页面中的相关词项进行注解说明。脚注的标记需要先用方括号和其中的上箭头和数字或者字母来引用，然后随后再定义这个脚注即可。至于脚注顺序号会自动生成，不需要可以排序。例如：

“横眉冷对千夫指，俯首甘为孺子牛。”这句诗出自鲁迅[^luxun]的《自嘲》，表达了他面对社会压力和批评时的无畏态度，同时也体现了他愿意为人民服务的奉献精神。
{: .code-example}

[^luxun]: 鲁迅（1881年9月25日—1936年10月19日），原名周樟寿，后改名周树人，字豫山，后改字豫才，浙江绍兴人。中国著名文学家、思想家、革命家、教育家、美术家、书法家、民主战士，新文化运动的重要参与者，中国现代文学的奠基人之一。鲁迅怀着现实关切，创造性地从中华优秀传统文化中汲取营养，创作出具有中国作风、中国气派的文学作品。被誉为“二十世纪东亚文化地图上占最大领土的作家”。

实现方式：

~~~markdown
“横眉冷对千夫指，俯首甘为孺子牛。”这句诗出自鲁迅[^luxun]的《自嘲》，表达了
他面对社会压力和批评时的无畏态度，同时也体现了他愿意为人民服务的奉献精神。
{: .code-example}

豫山，后改字豫才，浙江绍兴人。中国著名文学家、思想家、革命家、教育家、美术
家、书法家、民主战士，新文化运动的重要参与者，中国现代文学的奠基人之一。鲁
迅怀着现实关切，创造性地从中华优秀传统文化中汲取营养，创作出具有中国作风、
中国气派的文学作品。被誉为“二十世纪东亚文化地图上占最大领土的作家”。
~~~

### 1.1.1.5 缩略词

当你添加了一个缩略语定义后，当前页面所有的缩略语不需要任何修改，马上生效。例如：

这是一个 HTML 示范。

*[HTML]: 超级文本标记语言（Hyper Text Markup Language）的缩写

实现方式：

~~~ markdown
这是一个 HTML 示范。

*[HTML]: 超级文本标记语言（Hyper Text Markup Language）的缩写
~~~

### 1.1.1.6 HTML 元素

HTML 不仅支持块级元素也支持字级元素：

这是<span style="color: red">用红色书写</span>的。

实现方式：

~~~markdown
这是<span style="color: red">用红色书写</span>的。
~~~

### 1.1.1.7 行内属性

如同块级元素你也可以给字级元素设定一个字级行内属性列表（span inline attribute list——简写为 span IAL）。一个 span IAL 使用类同 block IAL 的语法，同时需要紧随字级元素：

这是**红色**{: style="color: red"}。

实现方式：

~~~ markdown
这是**红色**{: style="color: red"}。
~~~

## 1.1.2 块级标记

块级元素是构成内容结构框架的主要元素。

### 1.1.2.1 段落

用空行隔开的文字就形成了一个自然段落。

这是另一个段落。

实现方式：

~~~markdown
用空行隔开的文字就形成了一个自然段落。

这是另一个段落。
~~~

{: .note}
中文的段落开始文字通常会退两个字来显示段落的开始。这需要通过 CSS 来重新定义段落显示。

通常，一段文字就是一个段落。当我们在大屏幕上编辑排版的时候，对于过长的文字我们可能会习惯于通过回车将较长的文字截断，然后重现开始一行。这种截断，在转换之后，会用一个**空
格**将上下行连接起来。对于英文等以空格隔开单词的西文来说非常合理，但是对于中文就会多出一个空格。所以，如果是中文编辑，就不要将长文字通过回车来截断了。

如果需要在一段文字中截断，使其在视觉上达到回行的效果，可以通过在文字的末尾加上两个空格或者两个反`\\` 来生成 HTML 中的 `<br >` 标签完成在段落内部回行。例如：

黄河远上白云间，  
一片孤城万仞山。\\
羌笛何须怨杨柳？  
春风不度玉门关！

实现方式：

~~~markdown
黄河远上白云间，  
一片孤城万仞山。\\
羌笛何须怨杨柳？  
春风不度玉门关！
~~~

### 1.1.2.2 标题

标题使用 `#` 来完成，几个 `#` 就是几级标题。

~~~markdown
# H1 header

## H2 header

### H3 header

#### H4 header

##### H5 header

###### H6 header
~~~

生成标题时，会自动生成标题的 `id`，如果不想生成，可以将其属性 `auto_ids` 设置为 `false` 即可。

### 1.1.2.3 块引用

块引用以 `>` 标记开始，后面格这一个空格；后面所有以块引用标记开始的行都同属于一个块应用。块引用内部可以使用任何块级元素：

> A sample blockquote.
>
> >Nested blockquotes are
> >also possible.
>
> ## Headers work too
> This is the outer quote again.

实现方式：

~~~markdown
> A sample blockquote.
>
> >Nested blockquotes are
> >also possible.
>
> ## Headers work too
> This is the outer quote again.
~~~

也可以使用偷懒方式——开始用 `>`，后面在使用空行之前的所有部分都是块引用：

> 这是块引用，接下来
这行也是，
这行业也是，直到遇到空行

这行就不是块引用的部分了。

实现方式：

~~~markdown
> 这是块引用，接下来
这行也是，
这行业也是，直到遇到空行

这行就不是块引用的部分了。
~~~

### 1.1.2.4 代码段

两种方式可以实现代码段展示。一种是每行开始使用四个空格或者一个 <keyboard>Tab</keyboard>，另一种是使用 `~` 做为分割符，这种就不需要缩进了。例如第一种：

这是一段代码段。

    puts 'hello, world!'

实现方式：

~~~ markdown
这是一段代码段。

    puts 'hello, world!'
~~~

另一种：

~~~~
另一种方式的代码段。
~~~
结束行的符号一定不能少于开始行。
~~~~

实现方式：

~~~~~~markdown
~~~~
另一种方式的代码段。
~~~
结束行的符号一定不能少于开始行。
~~~~
~~~~~~

代码段可以指定代码的语言：

~~~ ruby
def what?
  42
end
~~~

实现方式：

~~~~markdown
~~~ ruby
def what?
  42
end
~~~
~~~~

### 1.1.2.5 水平线

插入水平线很容易，只需要三个以上的星号（*）、连接线（-）、或者下划线（_），甚至中间可以加上空格或者 <keyboard>Tab</keyboard> 的行，下面还有一个空行就会形成一条水平线。例如：

* * *

---

  _  _  _  _

---------------

实现方式：

~~~ markdown
* * *

---

  _  _  _  _

---------------
~~~

kramdown 支持有序列表和无序列表。有序列表始于一个数字，然后是一个点，一个空格，然后是列表项文字。列表项内容可包括块级元素。所有具有相同的缩进的带有列表项标记的行为同一列表项。

例如：

1. 这是一个列表项
2. 这是另一个列表项
2. 这是第三个列表项
   目的内容

实现方式：

~~~
1. 这是一个列表项
2. 这是另一个列表项
2. 这是第三个列表项
   目的内容
~~~

就像块引用一样，你可以使用偷懒方式使用列表项标记：

* 一个带有附加内容
的列表项

实现方式：

~~~
* 一个带有附加内容
的列表项
~~~

当内容包含块级元素时你可以这样做：

1. 这是一个列表项

   > 带有一个块引用

   # 和一个标题

2. 另一个列表项

实现方式：

~~~
1. 这是一个列表项

   > 带有一个块引用

   # 和一个标题

2. 另一个列表项
~~~

嵌套列表也很容易实现：

1. 列表一
   1. 子列表一
   2. 子列表二
   3. 子列表三
1. 列表二

实现方式：
~~~
1. 列表一
   1. 子列表一
   2. 子列表二
   3. 子列表三
1. 列表二
~~~

如果列表跟在其他块级元素的后面可能会连在一起，除非至少有一个空行隔开。例如段落后面的列表：

~~~
这是一个段落。
1. 这不是一个列表。

1. 这是一个列表！
~~~

显示如下：

这是一个段落。
1. 这不是一个列表。
1. 这是一个列表！

看起来，这个问题解决了！***段落后面跟着列表，会优先显示列表***。

无序列标是一个星号或者横线或者加号（三者也可以混用）后面再跟一个空格构成。其他规则同有序列表：

* 列表项目一
+ 列表项目二
- 列表项目三

实现方式：

~~~
* 列表项目一
+ 列表项目二
- 列表项目三
~~~

### 7 定义类表

定义列表工作方式类似于普通列表，主要用于关联术语和定义。定义列表开始像是普通的段落一样，但是接下来的行要以冒号开始，然后就是概念的解释内容。一个术语可能有多种解释，多个术语也可以有相同的解释。段落的前面每行都是一个术语。例如：

术语
: 定义
: 另一个定义

另一个术语
还有一个术语
: 术语的定义解释

实现方式：

~~~
术语
: 定义
: 另一个定义

另一个术语
还有一个术语
: 术语的定义解释
~~~

如果在定义前插入一个空行（注：在术语和定义之间有且仅有一个空行），定义会被一个段落包裹：

术语

: 定义
: 定义

实现方式：

~~~
术语

: 定义
: 定义
~~~

每个术语可以使用行内元素定制样式，每个定义可以被解析为块级元素。例如，你可以在定义中使用任何块级元素。只需要让定义的行使用相同的缩进即可：

这*是*一个术语

: 这很是一个段落

  > 一个块饮用

  # 一个标题
  
实现方式：

~~~
这*是*一个术语

: 这很是一个段落

  > 一个块饮用

  # 一个标题
~~~

### 8 表格

kramdown 支持使用语法创建简单表格。表格的每行始于一个管道符 `|`。如果管道符紧跟着一个连接线 `-`，一个分割行就创建了。分割行用于分割表头和表体（可选对齐表格栏），分割表体成多个部分等。如果管道符跟着等号 `=`，下面的表格行就是表脚了。

例如：

|一个|表格|
|有两|行|

实现方式：

~~~
|一个|表格|
|有两|行|
~~~

再如：

| Header1 | Header2 | Header3 |
|:--------|:-------:|--------:|
| cell1   | cell2   | cell3   |
| cell4   | cell5   | cell6   |
|----
| cell1   | cell2   | cell3   |
| cell4   | cell5   | cell6   |
|=====
| Foot1   | Foot2   | Foot3
{: rules="groups"}

实现方式：

~~~
| Header1 | Header2 | Header3 |
|:--------|:-------:|--------:|
| cell1   | cell2   | cell3   |
| cell4   | cell5   | cell6   |
|----
| cell1   | cell2   | cell3   |
| cell4   | cell5   | cell6   |
|=====
| Foot1   | Foot2   | Foot3
{: rules="groups"}
~~~

### 9 HTML 元素

karmdown 允许使用块级 HTML 标签（div，p，pre……）标记整块文字——只需使用块级 HTML 标签开始即可。kramdown 语法一般不会处理 HTML 标签内部的参数，除非使用 `parse_block_html` 参数。如果该参数设置为 `true`，那么块级 HTML 标签的内容不管是块级和是行内级都将被 karamdown 解析：

<div style="float: right">
Something that stays right and is not wrapped in a para.
</div>

{::options parse_block_html="true" /}

<div>
This is wrapped in a para.
</div>
<p>
This can contain only *span* level elements.
</p>

实现方式：

~~~
<div style="float: right">
Something that stays right and is not wrapped in a para.
</div>

{::options parse_block_html="true" /}

<div>
This is wrapped in a para.
</div>
<p>
This can contain only *span* level elements.
</p>
~~~

### 10 块属性

可以对块级元素设定属性。直接在块的后面添加块行内属性列表（IAL）。一个块的 IAL 包括一个左花括号，然后是一个冒号，然后是属性设定，然后是右花括号。看一个设定块引用的 `title` 属性的例子：

> 一个帅气的块引用
{: title="块引用的标题"}

实现方式：

~~~
> 一个帅气的块引用
{: title="块引用的标题"}
~~~

作为常见的想要设置更多的 CSS 类到一个元素的方式，这就是快捷方式：

> 帅气的块引用
{: .class1 .class2}

实现方式：

~~~
> 帅气的块引用
{: .class1 .class2}
~~~

也有设置 ID 的快捷方式，只需在 ID 前加一个井号即可：

> 帅气的块引用
{: #with-a-id}

实现方式：

~~~
> 帅气的块引用
{: #with-a-id}
~~~

有时我们想要对不同的元素使用相同的属性。kramdown 允许在一个地方定义属性列表（ALD）而在其他元素中引用该属性列表。ALD 与 IAL 具有类似的结构，但是冒号被冒号+引用名+冒号取代了。通过在一个块使用引用名就可以起到包含了一个属性列表的作用：

{:refdef: .c1 #id .c2 title="title"}
paragraph
{: refdef}

实现方式：

~~~
{:refdef: .c1 #id .c2 title="title"}
paragraph
{: refdef}
~~~

一个块的 IAL 或者 ALD 的顺序是很重要的，因为后定义的属性会覆盖前面定义的属性：

{:refdef: .c1 #id .c2 title="title"}
paragraph
{: refdef .c3 title="t" #para}

实现方式：

~~~
{:refdef: .c1 #id .c2 title="title"}
paragraph
{: refdef .c3 title="t" #para}
~~~

### 11 扩展

kramdown 通过一些常用的语法提供了一些很少用的功能。这允许需要的时候可以使用这些扩展功能。当前有些扩展可以忽略文字（类似于注释），或者插入输出 kramdown 选项设置的文字。

这有一个显示插入注释的示例：

This is a paragraph
{::comment}
This is a comment which is
completely ignored.
{:/comment}
... paragraph continues here.

Extensions can also be used
inline {::nomarkdown}**see**{:/}!

实现方式：

~~~
This is a paragraph
{::comment}
This is a comment which is
completely ignored.
{:/comment}
... paragraph continues here.

Extensions can also be used
inline {::nomarkdown}**see**{:/}!
~~~

如上所述，扩展语法类同于 ALD。然而扩展名没有终止冒号，而且扩展结束标签在扩展名字和冒号之间需要一个反斜线。当然也可以使用结束标签的简化形式，如`{:/}`。属性定义可以用空格同将属性名和开始标签分开。如果扩展没有主体，也可以在右侧使用一个斜线关闭括号：


{::options auto_ids="false" /}
# Header without id

实现方式：

~~~
{::options auto_ids="false" /}
# Header without id
~~~
