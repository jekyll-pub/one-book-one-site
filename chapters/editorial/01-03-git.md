# 1.3 Git 基础

存储对于文献的作者而言很重要。如果稍有不慎，编辑稿件丢失，则可能功亏一篑，前功尽弃。

所以，不管我们的作品进展如何，一定要做好存储。

除了存储，我们搞创作还有一个需要操心的事情就是作品的修改与版本问题。有些时候，我们的作品，不断的修改修改，如果一只将最后的版本做为自己的最好版本还可以，如果一旦想要将作品恢复到以前的某个时候就麻烦了。创作几乎就是格单程票，一直向前，不能回头了。还有些时候，我们在家里做了修改，但是没有将其带在身上，等到了工作室想要继续修改创作的时候，就发现单位的作品没有与家里的最新作品同步，此时要么在工作室重新做家里已经做完的工作，将作品重新修改一遍，要么回家拷贝作品过来再工作！所以，作品版本的管理问题也很重要。

## 1.3.1 本地存储

我们将作品存储到计算机硬盘上就是本地存储。

## 1.3.2 移动存储

我们将作品存储到移动硬盘、优盘等移动存储设备上就是移动存储。移动存储方便，但是可能保险系数不如本地存储。因为更多的操作可能导致文件误操作或者病毒等恶性程序入侵导致文件丢失。

## 1.3.3 云存储

云存储就是将作品保存的远程服务器上。远程服务器可以购买也可以使用免费服务。
